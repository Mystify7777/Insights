JavaScript Basics

JavaScript is a high-level, interpreted scripting language primarily used for client-side web development. It enables interactive web pages and is an essential part of web applications.

It is also used in server-side programming with Node.js, and in mobile applications with frameworks like React Native.


// Sample JavaScript: Hello World
function greet(name) {
  console.log(`Hello, ${name}!`);
}
greet("World");

// Sample JavaScript: Array Iteration
const numbers = [1, 2, 3, 4, 5];
numbers.forEach(num => console.log(num * 2));
  • Dynamically update HTML and CSS.
  • Implement interactive features like forms and animations.
  • Handle events (clicks, key presses).
  • Make asynchronous requests (AJAX, Fetch API).
HTML & CSS Fundamentals

HTML (HyperText Markup Language) is the standard markup language for documents designed to be displayed in a web browser. It defines the structure and content of web pages.

CSS (Cascading Style Sheets) is a style sheet language used for describing the presentation of a document written in HTML. CSS describes how elements should be rendered on screen, on paper, or in other media.


<!-- Sample HTML: Basic Structure -->
<!DOCTYPE html>
<html>
<head>
  <title>My Page</title>
</head>
<body>
  <h1>Welcome</h1>
</body>
</html>

/* Sample CSS: Styling a Heading */
h1 {
  color: #336699;
  font-family: sans-serif;
  text-align: center;
}
  • HTML provides the semantic structure of web content.
  • CSS controls layout, colors, fonts, and visual effects.
  • Responsive design is achieved using CSS media queries.
  • Modern CSS features include Flexbox, Grid, and custom properties.
Python for Web

Python is a versatile, high-level programming language known for its readability and extensive libraries. While not a browser-native language, Python is widely used for server-side web development.

Popular web frameworks like Django and Flask facilitate rapid web application development with Python.


# Sample Python: Flask Web App
from flask import Flask

app = Flask(__name__)

@app.route('/')
def hello_world():
    return 'Hello, World from Flask!'

if __name__ == '__main__':
    app.run(debug=True)
  • Used for backend logic, APIs, and database interactions.
  • Strong community support and vast ecosystem of libraries.
  • Excellent for data processing and machine learning integration.
  • Supports various deployment options (e.g., Heroku, AWS).
Ruby on Rails Overview

Ruby on Rails, often shortened to Rails, is a server-side web application framework written in Ruby. It follows the Model-View-Controller (MVC) architectural pattern.

Rails emphasizes convention over configuration, promoting rapid development and ease of maintenance.


# Sample Ruby on Rails: Controller Action
class ArticlesController < ApplicationController
  def index
    @articles = Article.all
  end

  def show
    @article = Article.find(params[:id])
  end
end
  • Convention over Configuration (CoC) philosophy.
  • Don't Repeat Yourself (DRY) principle.
  • Built-in ORM (Active Record) for database interaction.
  • Strong focus on developer productivity.
React & Angular Comparison

React and Angular are two of the most popular JavaScript frameworks for building single-page applications (SPAs) and complex user interfaces. While both serve similar purposes, they have distinct philosophies and approaches.

React, developed by Facebook, is a library for building UI components. Angular, maintained by Google, is a comprehensive framework with a more opinionated structure.


// Sample React Component
import React from 'react';

function WelcomeMessage(props) {
  return <h1>Hello, {props.name}</h1>;
}

// Sample Angular Component (TypeScript)
import { Component } from '@angular/core';

@Component({
  selector: 'app-root',
  template: '<h1>Hello, {{ name }}</h1>',
  styleUrls: ['./app.component.css']
})
export class AppComponent {
  name = 'Angular User';
}
  • React: Component-based, virtual DOM, JSX syntax, flexible.
  • Angular: Full-fledged framework, TypeScript, RxJS, CLI tools, opinionated.
  • Both support large-scale applications and active communities.
  • Choosing between them often depends on project requirements and team familiarity.
